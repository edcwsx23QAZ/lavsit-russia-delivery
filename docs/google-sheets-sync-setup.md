# Настройка синхронизации с Google Sheets

## Шаг 1: Создание Google Apps Script

1. Откройте вашу таблицу Google Sheets:
   https://docs.google.com/spreadsheets/d/1lP2s2eTYWBqdKVsymlfBxDRioP_AiGxlTWGhYjiqBXk/edit

2. Перейдите в меню: **Extensions → Apps Script**

3. Удалите весь код по умолчанию и вставьте код из файла `google-apps-script-code.gs`

4. Сохраните проект (Ctrl+S или Cmd+S)

## Шаг 2: Развертывание как веб-приложение

1. В редакторе Apps Script нажмите **Deploy → New deployment**

2. Нажмите на иконку шестеренки ⚙️ рядом с "Select type" и выберите **Web app**

3. Заполните форму:
   - **Description**: "Delivery CRM Sync"
   - **Execute as**: **Me** (ваш email)
   - **Who has access**: **Anyone** (важно для работы без авторизации)

4. Нажмите **Deploy**

5. При первом развертывании Google попросит авторизовать доступ:
   - Нажмите **Authorize access**
   - Выберите ваш Google аккаунт
   - Нажмите **Advanced → Go to [Project Name] (unsafe)**
   - Нажмите **Allow**

6. После успешного развертывания скопируйте **Web App URL** (выглядит как: `https://script.google.com/macros/s/.../exec`)

## Шаг 3: Настройка в приложении

1. Создайте файл `.env.local` в корне проекта (если его нет)

2. Добавьте строку:
   ```
   NEXT_PUBLIC_GOOGLE_APPS_SCRIPT_URL=https://script.google.com/macros/s/YOUR_SCRIPT_ID/exec
   ```
   Замените `YOUR_SCRIPT_ID` на URL, который вы скопировали на шаге 2

3. Перезапустите приложение (если оно запущено)

## Шаг 4: Проверка работы

1. Откройте приложение: http://localhost:9000/delivery-crm

2. Внесите любое изменение в заказ (например, измените адрес)

3. Подождите 3-5 секунд

4. Проверьте таблицу Google Sheets - данные должны синхронизироваться автоматически

5. Проверьте лист "История изменений" - там должны появляться записи обо всех изменениях

## Структура таблицы

### Лист "Доставки"
- **Строка 1**: Заголовки столбцов
- **Строки 2+**: Данные заказов

### Лист "История изменений" (создается автоматически)
- **Строка 1**: Заголовки
- **Строки 2+**: История всех изменений с timestamp

## Важные замечания

- Синхронизация происходит автоматически при каждом изменении данных
- Задержка синхронизации: 3 секунды (debounce)
- История изменений записывается сразу при каждом изменении
- Полная синхронизация происходит каждые 3 секунды после последнего изменения
- При удалении заказа в приложении он удаляется из таблицы
- При перемещении заказа (drag-and-drop) обновляется дата в таблице
- При группировке заказов информация о группе сохраняется в таблице

## Устранение проблем

### Синхронизация не работает
1. Проверьте, что URL правильно указан в `.env.local`
2. Проверьте консоль браузера на наличие ошибок
3. Убедитесь, что веб-приложение развернуто с доступом "Anyone"

### Ошибки авторизации
1. Убедитесь, что вы авторизовали доступ при развертывании
2. Попробуйте развернуть заново и авторизоваться снова

### Данные не обновляются
1. Проверьте, что скрипт правильно сохранен в Apps Script
2. Убедитесь, что ID таблицы в скрипте совпадает с вашей таблицей
